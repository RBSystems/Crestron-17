Crestron.Fusion.API.Entities NamespaceCrestron Fusion API Online Help
Contains entity classes used by Fusion API Server, which can be shared with client applications.
Classes
  ClassDescription
API_Action
Represents the action resource. The action resource is managed by the Fusion.API.Resources.ActionResource class. The resource manager will populate only ID, Name, and Description properties in the response for GET many operations.
API_ActionStep
Stores a representation of an action step. The action step object is contained in the API_Action class.
API_AlertRule
Stores a representation of an alert rule. The alert rule object is contained in the API_Attribute class.
API_AlertRuleCriteria
Stores a representation of criteria for an alert rule. The alert rule criteria object is contained in the API_AlertRule class.
API_Appointment
Represents the appointment resource. The appointment resource is managed by the Fusion.API.Resources.AppointmentResource class. Note that GET many operations will return fully populated entity representations.
API_AppointmentComparer
Provides a way to sort appointments by their start date/times
API_Asset
Represents the asset resource. The asset resource is managed by the Fusion.API.Resources.AssetResource class. The resource manager will only populate AssetID, AssetName, Make, Model, and SerialNumber properties in the response for GET many operations.
API_AssetCustomField
Stores a representation of an asset custom field. The asset custom field object is contained in the API_Asset class.
API_AssetType
API_Attribute
Represents the attribute resource. The attribute resource is managed by the Crestron.Fusion.API.Resources.AttributeResource class. The resource manager will fully populate all properties in the response for GET many operations.
API_Availability
Represents the resource of room availability. The availability resource is managed by the Fusion.API.Resources.AvailabilityResource class. The resource manager will only populate RoomID and Status properties in the response for GET many operations.

API_Location
Stores a representation of a location. The location object is contained in the API_Room class.
API_Node
Represents a tree node resource. The tree node resource is managed by the Crestron.Fusion.API.Resources.NodeResource The resource manager will traverse child nodes recursively and fully populate all properties in the response for GET many operations.
API_Person
Stores a representation of a person. The person object is contained in the API_Room.
API_Preset
Stores a representation of a preset. The preset object is contained in the API_Appointment object.
API_PresetField
Stores a representation of a preset field. The preset field is contained in the API_Preset class.
API_Processor
Stores a representation of a processor. The processor object is contained in the API_Room class.
API_Result
Contains status information and payload for outbound XML responses. The class is a response wrapper sent after both successful and failed resource operation requests. All properties are read only and only apply to resource operation responses. Each property has a default of null.
API_Role
Stores a representation of a role. The role object is contained in the API_Appointment class
API_RoleContact
Stores a representation of a role contact. The role contact object is contained in the API_Role class.
API_RoleOption
Stores a representation of a role option. The role option object is contained in the API_Role class.
API_Room
Represents the room resource. The room resource is managed by the Crestron.Fusion.API.Resources.RoomResource class. The resource manager will only populate RoomID, RoomName, and Description in the response for GET many operations.
API_RoomCategory
API_RoomCustomField
Stores a representation of a room custom field. The room custom field object is contained in the API_Room class.
API_Signal
Stores a representation of a signal. The signal object is contained in the API_Room object.
API_Symbol
Stores a representation of a symbol. The symbol object is contained in the API_Room class.

API_User
